# Community Feedback Loop

This project evolves from real user outcomes, not theory.

## Goal

Turn GitHub community feedback into weekly improvements to:
- Sprint playbook
- Prompt templates
- Service routing defaults

## Intake channels

1. GitHub Issues (`bug`, `playbook`, `win-report`, `failure-report`)
2. GitHub Discussions (strategy and market experiments)
3. Pull Requests (new templates, routing policies, integrations)

## Weekly triage workflow

1. Label and group all new reports
2. Rank by impact:
   - Revenue impact
   - Reproducibility
   - Ease of implementation
3. Ship the top 3 improvements in one weekly patch

## Required evidence for strategy feedback

- Market/channel
- Offer used
- Outreach count
- Reply rate
- Close rate
- Revenue result

## Optimization rubric

- Keep changes that improve close rate or delivery speed
- Revert changes that increase effort without measurable gain
- Document all major strategy changes in changelog notes
